# 类型定义完整说明（GameDataEditor）


### 0.2 类型配置结构：TypeConfig

```ts
export interface TypeConfigItem {
  base_type: string;            // 基础类型：int/float/string/struct/array/var
  type_render: string;          // 默认渲染组件（UI）
  default: any;                 // 默认值---->必须，其他的可有可无
  mem: string;                  // 备注/说明

  type_agv?: any;               // 默认参数（各类型含义不同）
  support_render?: string[];    // base_type 允许切换/支持的渲染器列表（白名单）
  struct_def?: any;             // 快捷结构体的字段定义（如 id_num）
  card_style?: any;             // 卡片展示样式（用于“卡片模式”如何展示该字段）
}

```

## 1. 类型配置 TypeConfig（系统内置类型表）
来源：`src/common/config/type_config.ts`

`TypeConfig` 定义每个内置 `type` 的默认行为（名称、基础类型、默认渲染、默认参数、默认值等）。典型字段：

- `name`：展示名
- `base_type`：基础类型（决定底层数据范畴）
- `type_render`：默认渲染组件（UI）
- `default`：默认值
- `mem`：说明/备注
- `type_agv`：默认参数（不同类型含义不同）
- `support_render`：**该基础类型允许切换/支持的渲染器列表**（白名单）
- `struct_def`：快捷结构体的字段定义（如 `id_num`）

---

## 2. 内置 type 列表（按类别）
来源：`TypeConfig` + 类型解析逻辑

### 2.1 基础类型
- `int` / `float` / `string` / `struct` / `array` / `var`

### 2.2 语义/快捷类型（映射到 base_type + type_render）
- 枚举：`enum_int`、`enum_string`
- 范围：`range_int`、`range_float`
- 标识：`id`、`ref_id`
- 资源：`img`、`snd`
- 其他：`color`、`date`、`bool`（0/1）

### 2.3 快捷结构体（Composite Struct）
- `id_num`、`id_string`、`string_num`、`img_num`、`snd_num`、`img_string`、`snd_string`

### 2.4 数组容器类型
- 写法：`array[子类型]`，例如 `array[int]`、`array[id_num]`

---

## 3. `type_agv` 参数详解（常用）
`type_agv` 通常为对象；部分输入场景可能为字符串，系统会在解析阶段做归一化。

### 3.1 枚举（enum_int / enum_string）
对象形态（推荐）：

- `type: 'enum_int'`
- `type_agv: { options: { '0': '普通:#CCCCCC', '1': '稀有:#00AAFF' } }`

UI 内部可能会把 `options` 归一为数组：`[{value,label,color}]`，编辑器保存时再转回对象。

### 3.2 范围（range_int / range_float）
- `type_agv: { min: 0, max: 100, step?: 1, percent?: false, decimal_places?: 2 }`

### 3.3 ID（id / ref_id）
- `type_agv: { ref_column: 'name' }`：用于“ID 显示关联哪一列文本”

### 3.4 资源（img / snd）
默认示例（TypeConfig 内置）：
- `img.type_agv: { suffix: '.png,.jpg,.jpeg,.bmp', path_mode: 'project' }`
- `snd.type_agv: { suffix: '.mp3,.wav,.ogg', path_mode: 'project' }`

### 3.5 结构体（快捷结构体）
快捷结构体（如 `id_num`）来自 `TypeConfig[结构体名].struct_def`，并用于 struct 渲染器按字段逐位渲染。

## 4. 最小示例（TypeConfig 项长什么样）
```jsonc
{
  "enum_int": {
    "base_type": "int",
    "type_render": "enum",
    "default": 0,
    "mem": "枚举(整型)",
    "type_agv": { "options": { "0": "普通", "1": "稀有" } },
    "support_render": ["input", "enum", "json"]
  },
  "id_num": {
    "base_type": "struct",
    "type_render": "struct",
    "default": [0, 0],
    "mem": "ID+数量",
    "struct_def": {
      "id": { "type": "ref_id", "default": 0, "mem": "引用ID" },
      "num": { "type": "int", "default": 0, "mem": "数量" }
    }
  }
}
```

## 5. card_style（卡片展示样式）

`card_style` 用于定义：当表格数据以“卡片模式”展示时，**某个字段应该如何在卡片上呈现**。

设计目标：
- 不需要写业务代码，仅通过配置就能把 `name/icon/...` 等字段在卡片中排成一个好看的布局
- 同一个字段可在不同表中被覆盖（表的 `struct_def` 内覆盖 `card_style`）
- 可与 `type_render/type_agv` 解耦：`card_style` 只负责“卡片布局与呈现”，不影响表格编辑器的渲染器选择

### 5.1 card_style 数据结构（建议）

```jsonc
"card_style": {
  "type_render": "img",               // 可选。为空则使用该字段最终 type_config.type_render
  "visible": true,                    // 可选。false 则卡片不显示该字段
  "layer": 0,                         // 可选。渲染层级，数字越大越靠上

  "frame": {
    "anchor": "tl",                   // tl/tr/bl/br/c（卡片坐标系锚点）
    "x": 12,                          // 坐标（默认单位 px）
    "y": 12,
    "w": 64,                          // 尺寸（可选；未填由组件自适应）
    "h": 64,
    "unit": "px"                      // px 或 percent（percent 表示 0..1 相对卡片尺寸）
  },

  "align": {
    "h": "left",                      // left/center/right（组件内部内容对齐）
    "v": "top"                        // top/middle/bottom
  },

  "text": {
    "variant": "title",               // title/body/caption（仅文本类字段常用）
    "clamp": 1                        // 行数截断（可选）
  },

  "image": {
    "fit": "cover",                   // cover/contain（仅图片类字段常用）
    "radius": 10                      // 图片圆角（可选）
  }
}
```

说明：
- `frame` 是核心：提供“位置/尺寸/锚点/单位”，保证任何字段都能被放到卡片上合适位置
- `type_render` 允许卡片层覆盖渲染器（例如同一字段在表格编辑用 input，但在卡片上希望用 badge/enum 标签样式）
- `layer` 用于处理遮挡关系（图标在底、标题在上、角标最上等）

### 5.2 推荐用法（项目级 type_config）
通常把 `name/icon/...` 这类通用字段做成项目级 `type_config`，并配置 `card_style`：
- 卡片列表 panel 若在表的 `struct_def` 中存在这些字段，则自动显示
- 若某个表想改卡片表现，可在该表的 `struct_def` 内覆盖该字段的 `card_style`
