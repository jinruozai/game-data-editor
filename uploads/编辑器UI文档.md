# 编辑器 UI 文档

本编辑器基于 `https://github.com/jinruozai/EditorFrame` 实现。

EditorFrame 的理念：
- 只需要实现各种业务 **Panel**
- 通过 **Dock** 系统组织/停靠/折叠/分栏这些 Panel，即可形成完整编辑器

---

## 1. 默认布局（Layout）

布局由四个区域组成：上 / 中（左-中-右三列）/ 下。

### 1.1 上（Top）
**编辑器工具栏（单行横条）**
- 最左：项目图标 + 项目名称
  - 点击弹出菜单：
    - 新建
    - 打开目录
    - 打开最近（子菜单）
    - 保存
    - 另存为
    - 导入
    - 导出 zip

### 1.1.1 项目加载规则

项目目录没有强制根结构要求，加载逻辑以“尽量打开”为原则：
- `gamedata.json` 是可选文件；存在时读取 `project` / `type_config` / `card_styles`，不存在时使用空项目配置与默认 card style。
- 遍历目录内所有 `.json` 文件。
- `gamedata.json` 之外，只有包含 `_table` 的 JSON 会作为表文件加载；其他 JSON 忽略。
- 表路径等于文件路径去掉 `.json` 后的相对路径，例如 `data/items.json` 加载为 `data/items`。
- 表文件格式：
  - `_table.struct_def`：表结构定义
  - `_table.card_style`：该表使用的 card style key，缺省为 `default`
  - 其他顶层 key：实体 ID，对应值为实体数据
- 如果 `struct_def` 里引用的 type 不在当前 `type_config`（含内置基础类型）中，加载不中断，只写入 error log。
- 坏 JSON、无 `_table` 的 JSON、重复 ID 等错误都不阻止项目打开；能识别的数据正常加载，识别不了的部分跳过。

### 1.2 中（Middle：三列）

#### 1.2.1 左侧（Left Dock：可折叠）
左侧为 Dock 区域，使用 **图标 Tab** 切换 Panel（Tab 按钮为图标，工具栏放右侧），支持折叠收起。

包含的 Panel：

1) **TableMap 列表 Panel（核心导航）**
- 顶部：`+` 按钮（添加新表）
- 列表：显示 `tableMap` 中的表（相对路径名）
- 每条右键菜单：
  - 修改表的 `struct_def`
  - 表改名
  - 删除表

2) **全局搜索 Panel**
- 在 `gameData` 上按 ID/字段值检索
- 搜索结果可定位到表与实体

3) **项目 TypeConfig 配置 Panel**
- 列表：展示项目 `project_type_config` 中的自定义 type
- 支持：添加 / 编辑 / 删除
- 删除 type 时必须检查引用：
  - 若存在引用，弹出提示：是否强制删除
  - 若强制删除，需要把对应表的 `struct_def` 中引用该 type 的字段定义删除，并清理相关数据（以值归一化规则处理）
- 添加 type：弹出“新 type 编辑面板”（可复用同一套 TypeConfigItem 编辑器）

#### 1.2.2 中间（Center Dock：主视图）
主视图为可关闭的 Tab（类似文件标签页）。

默认打开的内容来自左侧 TableMap 点击：
- 点击某个表 → 在主视图打开“表格数据 Panel”

主视图常见 Panel：
1) **表格数据 Panel（核心工作区）**
- 展示某个表（`tableMap[pathKey]`）的实体数据
- 顶部工具栏（Panel 内工具栏）：
  - **添加**：添加卡片/数据按钮
  - **删除**：当存在选中卡片时显示删除按钮
  - **显示模式切换**：卡片 / 列表 两种模式
  - **卡片大小**：当处于“卡片模式”时，显示一个可拖拽的卡片尺寸调节条（影响卡片网格列数/密度）
  - **排序下拉**：
    - 第一个下拉：选择按 `struct_def` 里的哪个字段排序（字段名列表来自该表 `struct_def`）
    - 第二个下拉：升序 / 降序
- 卡片模式（Card）显示规则：
  - 自动收集该表 `struct_def` 中包含 `card_style` 的字段，并按 `card_style.layer` 从低到高绘制
  - 常见项目做法：把 `name/icon` 等通用字段定义为项目级 `type_config`，并配置 `card_style`；这样任意表只要有这些字段就会自然显示
  - 若表内 `struct_def` 覆盖了字段的 `card_style`，则卡片表现天然被重载
- 支持筛选/分页（可选）
- 选中某条数据后，联动右侧属性面板

##### 卡片网格的选择/框选/拖拽排序（必须支持）

1) 基础选择操作
- 单击空白背景：取消所有选中状态，清空选区
- 单击卡片：选中该卡片，并取消其他所有卡片的选中状态
- 框选（空白处按下拖动）：绘制矩形选框，**卡片中心点**落在选框内的卡片被选中

2) 组合键选择（Ctrl/Cmd / Shift）
- Ctrl/Cmd + 单击：切换该卡片选中状态（不影响其他卡片）
- Ctrl/Cmd + 框选：增量框选（保留已有选中，并把新选框内卡片追加到选中集合）
- Shift + 单击：区间连选  
  - 选中从“上一次主要点击的卡片”到“当前点击的卡片”之间的所有卡片  
  - 若期间没有按住 Ctrl/Cmd：会先清空之前选区再做区间选中

3) 卡片拖拽排序（Drag & Drop）
- 触发条件：按住卡片并移动超过 **5px** 才算启动拖拽（防止误触）
- 单选拖拽：按住一张未选中的卡片拖拽 → 先将其设为单选 → 跟随鼠标移动 → 释放后插入目标位置并完成排序
- 多选成组拖拽：已选多张时，拖拽任意一张已选卡片：
  - 鼠标抓住的卡片跟随鼠标
  - 其他已选卡片不需要都聚集到鼠标下，但在排序结果上作为一个整体（队列）移动并插入到目标区域
- Drop：松开鼠标后，卡片落入当前悬停的网格空位，完成重新排序

4) 边界情况：点击已选卡片的“延迟结算”
场景：已选多张卡片，用户想单选其中一张，但也可能是想拖拽整组。  
处理：
- MouseDown 在已选卡片上：先不清空选区
- 若拖动超过 5px：判定为“多选拖拽”
- 若未超过 5px 就 MouseUp：才结算为单选（清空其他选中，仅保留当前点击卡片）

#### 1.2.3 右侧（Right：属性面板 / Inspector）
右侧为固定属性编辑面板：
- 选中某个卡片后，显示该条实体的属性编辑（基于 `struct_def` + `type_config` 渲染）
- 当未选中实体时，显示空态提示

### 1.3 下（Bottom Dock：可折叠）
下方为 Dock 区域：
- 工具栏在下方
- 可折叠收起
- **没有关闭按钮**

包含的 Panel：
1) **Log 面板**
- 需要能清晰区分：普通输出 / 警告 / 报错
- 支持过滤与复制
- 单击记录不删除；只有 Clear 按钮清空日志
- 双击 log 记录可定位到表/ID/字段（若有位置信息）

---

## 2. 交互联动（必须满足）
- 左侧选择表 → 中间打开对应表视图 Tab
- 中间选择实体 → 右侧显示实体属性
- Log 定位 → 自动切换表/实体并高亮字段（如果记录包含 pathKey/id/field）

---

## 3. 主题与视觉一致性（必须满足）
所有自定义 UI 组件（包括各类 Panel、字段编辑器、弹窗、右键菜单、下拉等）必须支持并遵循 EditorFrame/宿主应用的主题配置：
- 颜色：前景/背景/边框/强调色/危险色等使用主题 token，不硬编码
- 字体与字号：使用主题提供的排版规范
- 间距与圆角：使用主题 spacing/radius
- 状态：hover/active/focus/disabled/error/warn 的视觉表现与主题一致

> 目标：换一套主题（暗色/亮色/高对比）时，无需改业务组件代码即可整体一致。
